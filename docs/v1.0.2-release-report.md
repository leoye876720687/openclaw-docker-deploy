# v1.0.2 Deployment Reliability Release Report

**Date**: 2026-04-02  
**Status**: ✅ Released locally and verified

---

## Summary

This release tightens the deployment workflow for new OpenClaw Docker agents.
The main goal was to remove startup race conditions and make verification output trustworthy.

---

## Changes

### 1. Readiness-based initialization

`scripts/deploy-agent.sh` no longer relies on a fixed startup sleep alone.
It now waits for the container to reach a runnable state before invoking initialization.

### 2. Verification timeout handling

`scripts/verify-deployment.sh` now treats timeout-related output as a failed conversation check.
This avoids false positives where a timed-out run was previously reported as successful.

### 3. Isolated container behavior documented

The README and skill docs now align with the current isolated deployment model.
By default, the init workflow keeps host configuration out of the container unless explicitly requested.
