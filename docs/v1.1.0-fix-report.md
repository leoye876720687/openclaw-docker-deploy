# v1.1.0 Container Initialization Fix Report

**Date**: 2026-03-30  
**Issue**: `EACCES: permission denied` when creating sessions directory  
**Status**: ✅ Fixed

---

## Problem

When deploying new OpenClaw Docker containers, agents failed with:

```
run error: Error: EACCES: permission denied, mkdir '/home/openclaw/.openclaw/.openclaw/agents/main/sessions'
```

### Root Cause

1. **Missing Directory Structure**: Container startup only creates basic directories
2. **Missing `sessions/` Subdirectory**: Required for storing session files
3. **Missing `auth-profiles.json`**: No API authentication configured
4. **Default Model Issue**: Containers default to `anthropic/claude-opus-4-6` without API key

### Affected Containers

- `openclaw-agent-4` - Newly created, missing initialization
- `openclaw-agent-5` - Deleted and recreated
- Any new container deployed via `docker run`

---

## Solution

### 1. Created `init-agent-container.sh` Script

**Location**: `scripts/init-agent-container.sh`

**Features:**
- Creates complete directory structure (`agent/`, `sessions/`)
- Copies `auth-profiles.json` and `models.json` from host
- Sets correct permissions (`openclaw:openclaw`, `600` for auth files)
- Configures default model to `qwen-portal/coder-model`

**Usage:**
```bash
./scripts/init-agent-container.sh <container-name> [auth-profiles-source] [models-source]
```

### 2. Updated `deploy-agent.sh`

Now automatically calls `init-agent-container.sh` after container startup:

```bash
# Deploy
docker run -d --name $NAME ...

# Initialize (NEW!)
bash "$SCRIPT_DIR/init-agent-container.sh" $NAME

# Verify
docker ps --filter "name=$NAME"
```

### 3. Default Model Configuration

Changed from Anthropic (requires API key) to Qwen (OAuth):

```bash
docker exec $CONTAINER openclaw config set agents.defaults.model qwen-portal/coder-model
```

## Testing

### Test 1: Fresh Container Deployment

```bash

docker run -d --name openclaw-test-agent \
  -p 18899:18789 \
  -e NODE_OPTIONS=--max-old-space-size=2048 \
  f6e213e2e22c

  ~/.openclaw/agents/agent-4/agent/auth-profiles.json \
  ~/.openclaw/agents/agent-4/models.json

# Restart to apply config

# Test chat

  --session-id test --message "你好"
```

**Result**: ✅ Chat works (OAuth token issue is separate)

## Lessons Learned

1. **Always Initialize Directories**: Don't assume container has correct structure
2. **Set Permissions Explicitly**: Docker volumes may have different ownership
3. **Default Model Matters**: Avoid dependencies on unavailable API keys
4. **Test Fresh Deployments**: Always test with new containers, not just existing ones
